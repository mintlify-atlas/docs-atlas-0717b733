---
title: "Quickstart"
description: "Get Fintacharts Asset Tracker running locally in under five minutes using Docker Compose."
---

<Note>
  This guide uses Docker Compose to start the API and a PostgreSQL 16 database together. No local .NET SDK or PostgreSQL installation is required.
</Note>

<Steps>
  <Step title="Prerequisites">
    Make sure the following are installed on your machine:

    - [Docker](https://docs.docker.com/get-docker/) (v24 or later recommended)
    - [Docker Compose](https://docs.docker.com/compose/install/) (v2 or later â€” included with Docker Desktop)

    Verify your installation:

    ```bash
    docker --version
    docker compose version
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/9asmodey6/Fintacharts.AssetTracker.git
    cd Fintacharts.AssetTracker
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in your Fintacharts credentials:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and set the following variables:

    | Variable | Description | Default / Example |
    |---|---|---|
    | `FINTA_DB_CONNECTION` | PostgreSQL connection string used by EF Core | `Host=postgres;Port=5432;Database=asset_tracker;Username=postgres;Password=postgres` |
    | `FINTA_BASE_URL` | Base URL for the Fintacharts REST API | `https://platform.fintacharts.com/` |
    | `FINTA_WSS_URL` | WebSocket endpoint for real-time price streaming | `wss://platform.fintacharts.com` |
    | `FINTA_USER` | Your Fintacharts account username | `your_username_here` |
    | `FINTA_PASS` | Your Fintacharts account password | `your_password_here` |
    | `FINTA_REALM` | Keycloak realm used for authentication | `fintatech` |
    | `FINTA_CLIENT_ID` | OAuth2 client identifier | `app-cli` |

    The `.env.example` file ships with sensible defaults for everything except `FINTA_USER` and `FINTA_PASS` â€” those must be replaced with your own credentials.

    <Warning>
      Never commit your `.env` file to version control. It contains sensitive credentials and is already listed in `.gitignore`.
    </Warning>
  </Step>

  <Step title="Launch with Docker Compose">
    Build and start both the API and the database:

    ```bash
    docker-compose up --build
    ```

    Docker Compose will:
    1. Build the `asset-tracker` API image from `Fintacharts.AssetTracker/Dockerfile`.
    2. Start `asset-tracker-postgres` (PostgreSQL 16) on port `5432`.
    3. Wait for the database health check to pass, then start the API on port `8080`.
    4. Apply Entity Framework Core migrations automatically on startup.

    Once you see log output similar to the following, the service is ready:

    ```
    asset-tracker  | info: Microsoft.Hosting.Lifetime[14]
    asset-tracker  |       Now listening on: http://[::]:8080
    ```

    <Tip>
      Open Swagger UI in your browser to explore the API interactively: [http://localhost:8080/swagger](http://localhost:8080/swagger)
    </Tip>
  </Step>

  <Step title="Seed instruments (required)">
    <Note>
      This step is mandatory. Calling `GET /api/assets` seeds the database with financial instruments from Fintacharts and fires the internal `InstrumentsSyncedEvent`, which triggers the background worker to open WebSocket subscriptions. The prices endpoint will return no data until this step is completed.
    </Note>

    Call the assets endpoint to initialize the system:

    ```bash
    curl -X GET http://localhost:8080/api/assets
    ```

    A successful response returns the list of synchronized instruments as JSON, for example:

    ```json
    {
      "assets": [
        {
          "id": "ad9e5345-4c3b-41b2-8efe-2a2d5e2e27ab",
          "symbol": "EUR/USD",
          "description": "Euro vs US Dollar",
          "kind": "forex",
          "provider": "oanda"
        }
      ]
    }
    ```

    After this call returns, the `PriceUpdateWorker` is actively streaming real-time ticks over WebSocket in the background.
  </Step>

  <Step title="Verify real-time prices">
    Fetch the latest cached price for all tracked instruments:

    ```bash
    curl -X GET http://localhost:8080/api/prices
    ```

    Example response:

    ```json
    {
      "prices": [
        {
          "instrumentId": "ad9e5345-4c3b-41b2-8efe-2a2d5e2e27ab",
          "bid": 1.08321,
          "ask": 1.08334,
          "last": 1.08328,
          "updatedAt": "2026-03-26T10:15:00.000Z"
        }
      ]
    }
    ```

    Repeat the request every few seconds to observe live price updates as WebSocket ticks arrive.

    You can also fetch historical 1-minute bars for any instrument using its `id`:

    ```bash
    curl "http://localhost:8080/api/assets/ad9e5345-4c3b-41b2-8efe-2a2d5e2e27ab/history?barsCount=10"
    ```
  </Step>
</Steps>
