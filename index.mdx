## How it works

<Steps>
  <Step title="Deploy with Docker Compose">
    Spin up the API and PostgreSQL database with a single command. No manual database setup required.
  </Step>
  <Step title="Seed instruments">
    Call `GET /api/assets` once to fetch financial instruments from Fintacharts and store them locally. This also triggers the real-time price subscription engine.
  </Step>
  <Step title="Stream live prices">
    The background worker maintains a WebSocket connection to Fintacharts and continuously updates bid, ask, and last prices for all tracked instruments.
  </Step>
  <Step title="Query market data">
    Use `GET /api/prices` for live quotes and `GET /api/assets/{id}/history` for historical OHLC bars — all served from the local cache and database.
  </Step>
</Steps>

## Key capabilities

<CardGroup cols={2}>
  <Card title="Real-time WebSocket streaming" icon="bolt">
    Subscribes to Fintacharts L1 price feeds and processes bid/ask/last ticks as they arrive, with automatic reconnection on instrument changes.
  </Card>
  <Card title="Two-layer caching" icon="database">
    In-memory `ConcurrentDictionary` for sub-millisecond reads backed by PostgreSQL UPSERT for persistence across restarts.
  </Card>
  <Card title="Reactive event bus" icon="arrows-rotate">
    An in-memory event bus coordinates between the REST API and background WebSocket worker — no manual service restarts needed.
  </Card>
  <Card title="Automatic token management" icon="key">
    Thread-safe token refresh with a built-in buffer ensures the service never makes API calls with expired credentials.
  </Card>
</CardGroup>
