---
title: "Environment variables"
description: "Configure Fintacharts credentials and database connection"
---

All configuration is passed to the application via environment variables. When running with Docker Compose, these are loaded from a `.env` file in the project root. When running outside of Docker, set them directly in your shell or deployment environment.

## Set up the `.env` file

<Steps>
  <Step title="Copy the example file">
    Run the following command from the project root:

    <Tabs>
      <Tab title="Linux / macOS">
        ```bash
        cp .env.example .env
        ```
      </Tab>
      <Tab title="Windows (PowerShell)">
        ```bash
        cp .env.example .env
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Fill in your credentials">
    Open `.env` and replace the placeholder values for `FINTA_USER` and `FINTA_PASS` with your actual Fintacharts account credentials. The remaining values can be left at their defaults.
  </Step>
</Steps>

<Warning>
  Never commit the `.env` file to version control. It contains credentials and is already listed in `.gitignore`.
</Warning>

## `.env.example` contents

```bash
# Copy this file, rename it to .env, and enter your own values
FINTA_DB_CONNECTION=Host=postgres;Port=5432;Database=asset_tracker;Username=postgres;Password=postgres

FINTA_BASE_URL=https://platform.fintacharts.com/
FINTA_WSS_URL=wss://platform.fintacharts.com
FINTA_USER=your_username_here
FINTA_PASS=your_password_here
FINTA_REALM=fintatech
FINTA_CLIENT_ID=app-cli
```

## Variable reference

### Database

<ParamField path="FINTA_DB_CONNECTION" type="string" required>
  PostgreSQL connection string used by Entity Framework Core.

  ```
  Host=postgres;Port=5432;Database=asset_tracker;Username=postgres;Password=postgres
  ```

  When running with Docker Compose, `Host=postgres` refers to the `postgres` service defined in `compose.yaml`. Change this if you connect to an external database.
</ParamField>

### Fintacharts API

<ParamField path="FINTA_BASE_URL" type="string" required>
  Base URL for the Fintacharts REST API. Trailing slash required.

  ```
  https://platform.fintacharts.com/
  ```
</ParamField>

<ParamField path="FINTA_WSS_URL" type="string" required>
  WebSocket URL used for real-time price subscriptions.

  ```
  wss://platform.fintacharts.com
  ```
</ParamField>

<ParamField path="FINTA_USER" type="string" required>
  Your Fintacharts account username.
</ParamField>

<ParamField path="FINTA_PASS" type="string" required>
  Your Fintacharts account password.
</ParamField>

<ParamField path="FINTA_REALM" type="string" default="fintatech">
  The Keycloak realm used for OAuth token requests.
</ParamField>

<ParamField path="FINTA_CLIENT_ID" type="string" default="app-cli">
  The OAuth client ID used when authenticating with Keycloak.
</ParamField>

## How variables map to `appsettings.json`

Docker Compose passes each environment variable to the `api` container with a name that follows the ASP.NET Core configuration key format. The application then binds these to the `ConnectionStrings` and `Fintacharts` sections defined in `appsettings.json`.

| `.env` variable | ASP.NET Core key | `appsettings.json` path |
| --- | --- | --- |
| `FINTA_DB_CONNECTION` | `ConnectionStrings__Default` | `ConnectionStrings.Default` |
| `FINTA_BASE_URL` | `Fintacharts__BaseUrl` | `Fintacharts.BaseUrl` |
| `FINTA_WSS_URL` | `Fintacharts__WssUrl` | `Fintacharts.WssUrl` |
| `FINTA_USER` | `Fintacharts__Username` | `Fintacharts.Username` |
| `FINTA_PASS` | `Fintacharts__Password` | `Fintacharts.Password` |
| `FINTA_REALM` | `Fintacharts__Realm` | `Fintacharts.Realm` |
| `FINTA_CLIENT_ID` | `Fintacharts__ClientId` | `Fintacharts.ClientId` |

<Note>
  ASP.NET Core uses the double-underscore (`__`) as a hierarchy separator in environment variable names. `Fintacharts__BaseUrl` maps to `Fintacharts.BaseUrl` in `appsettings.json`, and `ConnectionStrings__Default` maps to `ConnectionStrings.Default`. This is the standard convention for overriding nested configuration settings via environment variables in ASP.NET Core.
</Note>
